# PDFラベルメーカー

## プロジェクト概要
テキストを入力し、ラベル用紙に合わせたPDFを生成するWebアプリケーション。

## 技術仕様
- HTML + JavaScript（サーバー不要、ブラウザだけで動く）
- PDF生成にはjsPDFライブラリを使う（CDNから読み込む）
- 日本語フォントはNotoSansJPをCDNから読み込み、jsPDFに登録して使用する

## ラベルの仕様
- 用紙サイズ: A4（210mm × 297mm）
- 余白（上下左右）を設定できる（デフォルト: 各10mm）
- ラベルのサイズ（幅×高さ）を設定できる（デフォルト: 70mm × 37mm）
- ラベル間の余白を設定できる（デフォルト: 2mm）
- ラベル内余白（パディング）: 2mm
- 行数・列数は用紙サイズ・余白・ラベルサイズ・ラベル間余白から自動計算する
- 各ラベルの周囲に枠線（罫線）を描画する
- テキストを入力し、すべてのラベルに同じ内容を印字する

## テキスト表示仕様
- テキスト配置: 左上揃え
- テキストがラベル幅を超える場合は自動折り返しする
- 複数行テキストに対応する
- フォントサイズはユーザーが指定できる（デフォルト: 10pt）

## PDF出力仕様
- 生成したPDFはブラウザの新しいタブで表示する

## コーディング規約
- JavaScriptの変数名・関数名はキャメルケース（camelCase）を使う
- コメントは日本語で書く
- アプリ本体は1つのHTMLファイルにまとめる。自前の外部CSSや外部JSファイルは作らない
- ただし、ラベル用紙プリセットのデータは `labellist.js` に外部化してよい（`file://` で直接開けるよう `.json` ではなく `.js` とし、`const labelList = [...]` で定義して `<script src="labellist.js">` で読み込む）
- 依存ライブラリはCDNから読み込んでよい

## 作業ルール
- 変更前に必ずgit commitすること
- テストや機能を変更した際は必ず `npm run test:coverage` を実行し、カバレッジ **100%**（ステートメント、ブランチ、関数、行すべて）が維持されていることを確認すること
- テスト実行用に一時的に抽出される `index.js` はコミットに含めないこと
- `pdflabel/` は親リポジトリ `study-claude` 側で一元管理するため、この配下に個別の `.git` を作成しないこと
- 変更は親リポジトリ側から GitHub リモートへプッシュすること。これにより GitHub Actions が自動でトリガーされます

## 開発・検証コマンド
- **テストの実行**: `npm run test` (インラインJSを抽出してVitest + JSDOMで実行)
- **テストの継続監視**: `npm run test:watch` (ファイル変更時に自動再テスト)
- **カバレッジの検証**: `npm run test:coverage` (カバレッジ100%閾値が強制され、下回るとエラーになります)
- **HTMLの静的検証**: `npx htmlhint index.html`
- **CI/CDによるデプロイ先**: [https://katoy.github.io/study-claude/pdflabel/](https://katoy.github.io/study-claude/pdflabel/) (mainブランチへのプッシュで自動更新)
