# お問い合わせフォーム 仕様書

## 1. 概要

Laravelで構築されたお問い合わせフォームアプリケーション。
エンドユーザーが利用する「お問い合わせフォーム」と、管理者が利用する「管理ページ」の2つの機能で構成される。

## 2. 技術スタック

| 項目 | 技術 |
|---|---|
| フレームワーク | Laravel 12 |
| 言語 | PHP |
| データベース | SQLite |
| CSS | インラインCSS（Vite/Tailwind不使用） |
| ページネーション | Bootstrap 5スタイル |

## 3. データベース設計

### contacts テーブル

| カラム | 型 | 制約 | 説明 |
|---|---|---|---|
| id | bigint | PK, AUTO_INCREMENT | 自動採番 |
| name | string(255) | NOT NULL | 名前 |
| email | string(255) | NOT NULL | メールアドレス |
| subject | string(255) | NOT NULL | 件名 |
| body | text | NOT NULL | 本文 |
| status | string(255) | NOT NULL, DEFAULT '新規' | ステータス |
| created_at | timestamp | NULLABLE | 作成日時 |
| updated_at | timestamp | NULLABLE | 更新日時 |

### ステータスの種類

| 値 | 説明 |
|---|---|
| 新規 | 初期状態（デフォルト値） |
| 対応中 | 対応を開始した状態 |
| 解決済み | 対応が完了した状態 |

## 4. ルーティング

### お問い合わせフォーム

| メソッド | URL | ルート名 | 処理内容 |
|---|---|---|---|
| GET | `/` | contact.create | 入力フォーム表示 |
| POST | `/contact/confirm` | contact.confirm | 確認画面表示 |

| PATCH | `/admin/contacts/{contact}` | admin.contacts.update | ステータス更新 |

## 5. お問い合わせフォーム機能

### 5.1 画面遷移フロー

```
入力フォーム → 確認画面 → 送信処理 → 完了画面
                 ↓
              「戻る」→ 入力フォーム（入力値復元）
```

### 5.2 入力フォーム（GET `/`）

入力項目:

| 項目 | 入力形式 | 必須 |
|---|---|---|
| 名前 | テキスト | はい |
| メールアドレス | メール | はい |
| 件名 | テキスト | はい |
| 本文 | テキストエリア | はい |

- 確認画面から「戻る」で遷移した場合、セッションに保存された入力値が復元される
- バリデーションエラー時は `old()` により入力値が復元される

### 5.3 バリデーションルール

| フィールド | ルール | エラーメッセージ |
|---|---|---|
| name | required, string, max:255 | 名前を入力してください。 / 名前は255文字以内で入力してください。 |
| email | required, email, max:255 | メールアドレスを入力してください。 / 正しいメールアドレスの形式で入力してください。 / メールアドレスは255文字以内で入力してください。 |
| subject | required, string, max:255 | 件名を入力してください。 / 件名は255文字以内で入力してください。 |
| body | required, string | 本文を入力してください。 |

### 5.4 確認画面（POST `/contact/confirm`）

- バリデーション通過後、入力データをセッション（キー: `contact_data`）に保存して表示
- 表示項目: 名前、メールアドレス、件名、本文
- 操作ボタン:
  - **戻る**: 入力フォームにリダイレクト（セッションのデータで入力値を復元）
  - **送信する**: 送信処理へPOST

### 5.5 送信処理（POST `/contact`）

- 「戻る」ボタン（`back`パラメータ）が押された場合: 入力フォームにリダイレクト
- セッションから `contact_data` を取得してDBに保存
- 保存後、セッションから `contact_data` を削除
- セッションが切れている場合: 入力フォームにリダイレクト

### 5.6 完了画面（GET `/contact/thanks`）

- 「お問い合わせありがとうございました」のメッセージを表示
- 「トップに戻る」リンクで入力フォームに遷移

## 6. 管理ページ機能

### 6.1 一覧表示（GET `/admin/contacts`）

- お問い合わせを新しい順（`latest()`）で表示
- 1ページあたり10件のページネーション
- 表示カラム: ID、名前、件名、ステータス、受信日時
- 名前をクリックすると詳細画面に遷移
- ステータスはバッジで色分け表示:
  - 新規: 赤（`#e74c3c`）
  - 対応中: オレンジ（`#f39c12`）
  - 解決済み: 緑（`#27ae60`）
- データが0件の場合は「お問い合わせはまだありません。」を表示

### 6.2 詳細表示（GET `/admin/contacts/{contact}`）

- 表示項目: ID、名前、メールアドレス、件名、本文、受信日時
- ステータス変更用のドロップダウンリスト（新規 / 対応中 / 解決済み）
- 「ステータスを更新」ボタンで更新処理
- 「一覧に戻る」リンク
- ステータス更新成功時にフラッシュメッセージ「ステータスを更新しました。」を表示

### 6.3 ステータス更新（PATCH `/admin/contacts/{contact}`）

- バリデーション: `status` が `新規` / `対応中` / `解決済み` のいずれかであること
- 更新後、詳細画面にリダイレクト（成功メッセージ付き）

## 8. セキュリティ対策

| 対策 | 実装方法 |
|---|---|
| CSRF保護 | 全フォームに `@csrf` ディレクティブを使用 |
| XSS防止 | Blade の `{{ }}` による自動エスケープ、本文の改行表示は `{!! nl2br(e()) !!}` を使用 |
| データ改ざん防止 | 確認画面のデータはセッション方式で保持（隠しフィールド不使用） |
| マスアサインメント防止 | モデルの `$fillable` で許可カラムを明示的に指定 |

## 9. 認証

現在、管理ページに認証機能は実装されていない。`/admin/contacts` に直接アクセスすることで管理機能を利用できる。
