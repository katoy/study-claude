# ToDo アプリ 仕様書

## 1. 概要

| 項目 | 内容 |
|------|------|
| アプリ名 | TODOリスト |
| 目的 | ブラウザ上で動作するシンプルなタスク管理アプリ |
| 技術スタック | HTML + CSS + JavaScript（単一ファイル） |
| 外部ライブラリ | Quill（リッチテキストエディタ、CDN読み込み）、Google Fonts（CDN読み込み） |
| データ永続化 | localStorage |
| サーバー | 不要（静的ファイルのみ） |
| デザイン | Corporate Intelligent Dark Aesthetic（Glassmorphism、グラデーション） |

## 2. 技術要件

- **単一HTMLファイル**: CSS・JavaScriptをすべてインラインで記述する
- **サーバー不要**: ブラウザでHTMLファイルを直接開くだけで動作する
- **データ永続化**: `localStorage` を使用し、ブラウザを閉じてもデータが保持される
- **リッチエディタ**: Quill を CDN から読み込み、詳細欄に使用する
- **レスポンシブ対応**: 不要（デスクトップブラウザを想定）
- **フォント**: Google Fonts (Plus Jakarta Sans, Noto Sans JP) を CDN から読み込み
- **UI デザイン**: Corporate Intelligent Dark Aesthetic
  - Glassmorphism（ガラス質の透明効果）
  - グラデーション背景とテキスト
  - CSS Variables によるテーマ管理
  - ニューモルフィズム的なボタン・フォーム要素

## 3. データ構造

### 3.1 ToDoアイテム スキーマ

```json
{
  "id": "string (UUID v4)",
  "title": "string (最大100文字)",
  "detail": "string (HTML形式, Quillの出力)",
  "dateType": "none | date | datetime",
  "deadline": "string | null (ISO 8601形式)",
  "completed": "boolean",
  "createdAt": "string (ISO 8601形式)"
}
```

| フィールド | 型 | 説明 |
|-----------|------|------|
| `id` | string | UUID v4 で自動生成。一意識別子 |
| `title` | string | タスクのタイトル。最大100文字 |
| `detail` | string | 詳細内容。Quillが出力するHTML文字列。プレーンテキスト換算で最大2000文字 |
| `dateType` | string | 日時の種類。`"none"`=なし / `"date"`=日のみ / `"datetime"`=時間まで |
| `deadline` | string \| null | 締め切り日時。`dateType` が `"none"` の場合は `null` |
| `completed` | boolean | 完了状態。`true`=完了 / `false`=未完了 |
| `createdAt` | string | 作成日時。ISO 8601形式 |

### 3.2 localStorage キー設計

| キー | 値 | 説明 |
|------|-----|------|
| `todo-app-items` | JSON配列 | ToDoアイテムの配列 |

## 4. メイン画面仕様

ワイヤーフレーム（main.jpg）に基づく。

### 4.1 レイアウト構成

```
┌──────────────────────────────────┐
│ TODOリスト                       │
│ [新規]                           │
│                                  │
│ [すべて] [未完了] [完了済]        │
│                    [完済を削除]   │
│                                  │
│ ── 本日中 ──────────────────     │
│  タイトル  YYYY-MM-DD HH:MM  [完了]  │
│  タイトル  YYYY-MM-DD HH:MM  [完了]  │
│                                  │
│ ── 明日まで ────────────────     │
│  タイトル  YYYY-MM-DD HH:MM  [完了]  │
│                                  │
│ ── それ以外 ────────────────     │
│  タイトル  YYYY-MM-DD HH:MM  [完了]  │
│                                  │
└──────────────────────────────────┘
```

### 4.2 ヘッダー

- タイトル「TODOリスト」を表示
- **[新規]ボタン**: クリックで詳細画面（モーダル）を新規登録モードで開く

### 4.3 タブ（フィルター）

3つのタブで表示を切り替える:

| タブ | 表示対象 |
|------|---------|
| すべて | 全てのToDoアイテム |
| 未完了 | `completed === false` のアイテム |
| 完了済 | `completed === true` のアイテム |

- デフォルトは「すべて」を選択状態とする
- 選択中のタブは視覚的に区別する（アクティブスタイル）

### 4.4 [完済を削除]ボタン

- クリック時、確認ダイアログを表示:「完了済みのToDoをすべて削除しますか？」
- 「OK」で `completed === true` の全アイテムを削除し、localStorage を更新
- 「キャンセル」で何もしない

### 4.5 セクション分け

ToDoアイテムを以下の4セクションに分類して表示する:

| セクション | 条件 |
|-----------|------|
| 期限切れ | deadline が今日の 00:00:00 未満 |
| 本日中 | deadline が今日の 00:00:00 以上、23:59:59.999 以下 |
| 明日まで | deadline が明日の 00:00:00 以上、23:59:59.999 以下 |
| それ以外 | 上記に該当しない（deadline が明後日以降、または dateType === "none"） |

- 「今日」「明日」は現在のブラウザのローカル時刻基準で判定する
- 各セクションにアイテムがない場合、そのセクション自体を非表示とする

### 4.6 ソート順

各セクション内のソート規則:

1. `deadline` が設定されているアイテムを先に表示（`dateType !== "none"`）
2. `deadline` 昇順（早い順）
3. `deadline` が同じ、または `dateType === "none"` のアイテムは `createdAt` 昇順（古い順）

### 4.7 各行の表示

各ToDoアイテムは1行で以下を表示:

| 要素 | 説明 |
|------|------|
| タイトル | タスク名を表示。クリックで詳細画面を編集モードで開く |
| 日時 | `dateType` に応じた表示（後述） |
| [完了]ボタン | クリックで `completed` をトグル |

#### 日時の表示形式

| `dateType` | 表示 |
|-----------|------|
| `none` | 表示なし |
| `date` | `YYYY-MM-DD` (例: `2026-07-24`) |
| `datetime` | `YYYY-MM-DD HH:MM` (例: `2026-07-24 12:00`) |

#### 完了状態の表示

- 未完了: [完了]ボタンを表示
- 完了済: ボタンのスタイルを変更（例: グレーアウト、打ち消し線など）し、クリックで未完了に戻せる

## 5. 詳細画面仕様（モーダルポップアップ）

ワイヤーフレーム（detail.jpg）に基づく。

### 5.1 レイアウト構成

```
┌──────────────────────────────────┐
│                                  │
│  タイトル [________________] 0/100│
│                                  │
│  日時     ○ なし                 │
│           ○ 日のみ  [YYYY-MM-DD] │
│           ○ 時まで  [YYYY-MM-DD HH:MM] │
│                                  │
│  詳細     ┌────────────────┐     │
│           │ Quillエディタ   │     │
│           │                │ 0/2000
│           └────────────────┘     │
│                                  │
│  [保存]  [キャンセル]            │
│                                  │
└──────────────────────────────────┘
```

### 5.2 モーダル動作

- 背景をオーバーレイで暗くする
- モーダル外クリックでは閉じない（明示的に[キャンセル]を押す必要あり）
- ESCキーで[キャンセル]と同じ動作

### 5.3 タイトル入力

- テキスト入力フィールド
- 最大100文字
- 必須項目（空欄では保存不可）
- プレースホルダー: 「タイトルを入力」
- **リアルタイム文字数カウンター**: 入力欄の右下に「現在の文字数/100」を表示し、入力と連動して動的に更新される。100文字を超えた場合はカウンターを警告色（赤など）で強調する。

### 5.4 日時選択

ラジオボタン3択:

| 選択肢 | 動作 |
|--------|------|
| なし | 日時入力欄を非表示 |
| 日のみ | 日付入力欄を表示 |
| 時まで | 日時入力欄を表示 |

- デフォルトは「なし」を選択
- ラジオボタン切り替え時、入力欄の表示/非表示を即時反映
- **ロケール非依存の書式・カレンダー表示（オーバーレイ方式）**:
  - ブラウザの言語設定による日付表記の強制変更を防ぎ、常にハイフン区切り（`YYYY-MM-DD` / `YYYY-MM-DD HH:MM`）で初期値や選択値を表示するため、**読み取り専用のダミーテキストインプット（`<input type="text">`）と透明なネイティブインプット（`type="date"` / `type="datetime-local"`）を重ねて配置**している。
  - 入力欄をクリックすると、JavaScript の `showPicker()` API が起動し、ネイティブのカレンダーポップアップが表示される。
  - 右端にはカレンダー絵文字（`📅`）を絶対配置し、視覚的なインジケーターとしている。

### 5.5 詳細入力（Quillエディタ）

- Quill リッチテキストエディタを使用
- ツールバー: 太字、イタリック、下線、リスト（箇条書き・番号付き）
- プレーンテキスト換算で最大2000文字
- 任意項目（空欄でも保存可）
- **リアルタイム文字数カウンター**: エディタの右下に「現在の文字数/2000」を表示し、入力と連動して動的に更新される（プレーンテキスト換算、末尾改行を除く）。2000文字を超えた場合はカウンターを警告色（赤など）で強調する。

### 5.6 ボタン

| ボタン | 動作 |
|--------|------|
| [保存] | バリデーション後、データを保存してモーダルを閉じる |
| [キャンセル] | 変更を破棄してモーダルを閉じる |

### 5.7 新規登録・編集の共用

| モード | 動作 |
|--------|------|
| 新規登録 | 全フィールド空欄で開く。保存時に新しいアイテムを追加 |
| 編集 | 既存データをフィールドに反映して開く。保存時に該当アイテムを更新 |

## 6. 動作仕様の詳細

### 6.1 「期限切れ」「本日中」「明日まで」の境界ロジック

```
現在時刻: now
今日の始まり: today 00:00:00.000
今日の終わり: today 23:59:59.999
明日の終わり: tomorrow 23:59:59.999

期限切れ: deadline < 今日の始まり
本日中:   今日の始まり <= deadline <= 今日の終わり
明日まで: 今日の終わり < deadline <= 明日の終わり
それ以外: deadline > 明日の終わり OR deadline === null
```
- `dateType === "date"` の場合、`deadline` は指定日の `00:00:00` として扱う
  - 例: 「2026-03-15」指定 → `2026-03-15T00:00:00` → 3月14日時点では「明日まで」、3月15日時点では「本日中」、3月16日以降は「期限切れ」

### 6.2 完了フロー

1. メイン画面で[完了]ボタンをクリック
2. `completed` を `true` に更新
3. localStorage を保存
4. 画面を再描画（タブに応じてフィルタリング）

### 6.3 完了済みアイテムの未完了への復帰

1. 完了済みアイテムの[完了]ボタン（トグル）をクリック
2. `completed` を `false` に更新
3. localStorage を保存
4. 画面を再描画

### 6.4 削除フロー

- 個別削除機能はない
- [完済を削除]ボタンで完了済みアイテムを一括削除のみ

### 6.5 バリデーションとデータの正規化

保存時に以下をチェック:

| チェック項目 | 条件 | エラーメッセージ |
|------------|------|----------------|
| タイトル必須 | 空文字 or 空白のみ | 「タイトルを入力してください」 |
| タイトル文字数 | 101文字以上 | 「タイトルは100文字以内で入力してください」 |
| 詳細文字数 | プレーンテキスト2001文字以上 | 「詳細は2000文字以内で入力してください」 |
| 日付必須 | `dateType` が `date` or `datetime` で日付未入力 | 「日時を入力してください」 |

- エラーメッセージは該当フィールドの近くに赤字で表示
- **保存時の日付データ正規化**:
  - 各ブラウザの入力コントロールから取得される値（スラッシュ `/` 区切りやスペース区切りなど）のフォーマットのブレを排除するため、保存実行時に正規表現を用いて置換処理を行い、必ず `YYYY-MM-DDTHH:MM:00` または `YYYY-MM-DDT00:00:00` の厳格な ISO-8601 形式に正規化して `localStorage` に保存する。

## 8. セキュリティ仕様

### 8.1 入力とリッチテキストの安全性

**タイトル入力** (`title` フィールド):
- 最大100文字に制限
- ブラウザにより常に `textContent` で DOM に挿入（HTML エスケープ）
- `innerHTML` は使用しない

**詳細入力** (`detail` フィールド):
- Quill リッチテキストエディタで入力
- ツールバー機能: 太字、イタリック、下線、箇条書きリスト、番号付きリスト のみ
- 許可されるタグ: `<b>`, `<i>`, `<u>`, `<li>`, `<ol>` のみ
- Quill 内部で HTML をフィルタリング
- localStorage との往復時も Quill が処理を行い、不正な HTML は取り除かれる

### 8.2 localStorage のデータセキュリティ

- **データ格納場所**: ブラウザの localStorage（ローカルストレージ）
- **アクセス制限**: 同一オリジン内でのみアクセス可能（CORS 対象外）
- **データ改ざん検出**: アプリケーションレベルでの完全性チェックなし
  - localStorage の改ざんが疑われる場合、JSON パース失敗時は空配列にリセット
- **シークレット/プライベートモード**: ブラウザセッション終了時に localStorage はクリアされる可能性あり
- **サイズ上限**: 通常 5-10MB。このアプリの想定使用では問題なし

### 8.3 JSON パースと検証

```javascript
const loadTodos = () => {
  try {
    const stored = localStorage.getItem(STORAGE_KEY);
    if (stored) {
      const parsed = JSON.parse(stored);
      if (Array.isArray(parsed)) {
        todos = parsed;
      } else {
        todos = [];
      }
    } else {
      todos = [];
    }
  } catch (e) {
    todos = [];
  }
};
```

- `JSON.parse()` を try-catch でラップ
- パース失敗時は空配列にリセット
- 復元後のオブジェクトが配列であることを確認
- 個別 todo オブジェクトのスキーマ検証は行わない（パフォーマンス優先）

### 8.4 バリデーションと正規化

**保存時のバリデーション**:

| 項目 | チェック | 失敗時の処理 |
|------|---------|-----------|
| タイトル | 空文字・空白のみ | エラー表示、保存キャンセル |
| タイトル文字数 | 101文字以上 | エラー表示、保存キャンセル |
| 詳細文字数 | プレーンテキスト 2001 文字以上 | エラー表示、保存キャンセル |
| 日付 | `dateType` が `date`/`datetime` で未入力 | エラー表示、保存キャンセル |

**日付の正規化**:
- ブラウザネイティブの date/datetime-local 入力から得られる値は形式にばらつきがある
- 保存時に正規表現で置換処理を行い、常に `YYYY-MM-DDTHH:MM:SS` 形式（ISO-8601）に統一
- Quill からの HTML 出力も保存前にプレーンテキスト文字数をカウント

### 8.5 CDN と外部依存

**Quill エディタ**:
- CDN: `https://cdn.jsdelivr.net/npm/quill@2/...`
- バージョン: `@2`（最新マイナー・パッチを動的に取得）
- リスク: 新しいバージョンでセキュリティ脆弱性が見つかった場合、自動的に取り込まれる
- 推奨: 定期的にセキュリティアドバイザリを確認し、必要に応じてバージョンを固定

**Google Fonts**:
- CDN: `https://fonts.googleapis.com`, `https://fonts.gstatic.com`
- HTTPS でのみ読み込み
- フォール バック: CDN 障害時はシステムフォントにフォールバック
- プライバシー: Google Fonts の読み込みにより IP アドレスがログに記録される可能性

### 8.6 既知の前提と限界

- **ローカル実行の前提**: このアプリはローカルストレージ機能とローカル実行を前提としており、サーバーバックエンドやシステム間での認証・暗号化は存在しない
- **ブラウザセキュリティの信頼**: ブラウザの SOP（Same-Origin Policy）、CSP（Content Security Policy）などのセキュリティメカニズムに依存
- **XSS 対策**: 主として Quill 的な制限されたツールバーと `textContent` の使用に依存。追加のサニタイズライブラリは使用していない
